# Lookthrough 顯示門檻 QA Checklist

本 checklist 用來驗證「穿透分析」頁的顯示門檻與小額成分彙總。這些設定只影響底層股票曝險表格的列出方式，不應改變原始 lookthrough exposure、總市值、市場曝險、產業曝險或集中度計算。

## A. 小型台股 ETF 情境

測試資料：
- 持股包含 `0050` 與 / 或 `00981A`。
- ETF 成分股已透過 proxy 或 CSV 匯入。

檢查項目：
- 主要持股例如 `2330`、`2454`、`2308` 仍正常顯示。
- 成分市場顯示為「台股成分」。
- 預設門檻 `NT$10` / `0.01%` / 最多 `50` 筆下，正常大小的台股成分不應被誤彙總。
- 若手動提高門檻，低於門檻的台股成分會彙總為「其他台股成分」。
- 彙總列顯示已彙總的成分股數量。

## B. 00646 大型美股成分情境

測試資料：
- 持股包含 `00646`。
- 在「ETF 成分股」頁手動或 CSV 匯入多筆美股 ticker，例如 `AAPL`、`MSFT`、`NVDA`、`AMZN`、`META` 與其他小權重成分。
- 另測 Bloomberg-like ticker：`NVDA UQ`、`AAPL UQ`、`JPM UN`、`CBOE UF`，以及 class-share ticker：`BRK/B`、`BF/B`。

檢查項目：
- 美股 ticker 會分類為「美股成分」。
- Bloomberg-like suffix 會被清掉，例如 `NVDA UQ` -> `NVDA`。
- class-share slash 會轉為 dot，例如 `BRK/B` -> `BRK.B`。
- 大型持股仍逐列顯示。
- 很小的持股低於門檻時，會彙總為「其他美股成分」。
- 「其他美股成分」顯示已彙總的檔數。
- 超過最多顯示筆數的其餘美股 rows 也會彙總為「其他美股成分」。
- 彙總來源欄不應列出大量逐筆 source，只需顯示彙總摘要。
- 「市場曝險分類」仍使用完整原始資料，不因表格彙總而少算。

## C. 門檻邊界情境

測試項目：
- 最小顯示金額 = `0`，最小投組佔比 = `0`。
- 門檻設得非常高，例如金額 `200000`、佔比 `50%`。
- 取消勾選「將低於門檻的成分彙總為其他」。
- 全部 rows 都低於門檻。
- 沒有任何 rows 低於門檻。

預期結果：
- `0` / `0%` 時，所有成分應逐列顯示，不應產生彙總列。
- 高門檻時，小額 rows 依市場彙總。
- 取消勾選時，表格顯示完整明細，不產生「其他...成分」彙總列。
- 全部 rows 低於門檻時，仍應至少顯示對應市場的彙總列。
- 沒有 rows 低於門檻時，不應顯示彙總列。
- 負數或無效輸入應視為 `0`，不應造成畫面錯誤。

## D. 00646 尚未匯入成分股情境

測試資料：
- 持股包含 `00646`。
- `00646` 尚未匯入任何 ETF 成分股。

檢查項目：
- LookthroughPage 顯示警示：
  `00646 屬於海外成分股 ETF，目前尚未匯入美股成分股，會暫時以單一美股 ETF 曝險呈現。`
- `00646` 不應被分類為台股成分。
- 底層股票曝險表中，`00646` 可暫時作為單一美股 ETF placeholder。
- 匯入 `00646` 成分股後，未對應 ETF 警示中不應再列出 `00646`。

## E. 不應改變的行為

確認：
- `0050` / `00981A` proxy 更新流程仍可預覽與儲存。
- CSV / 手動匯入 fallback 仍可使用。
- MVP navigation 沒有改變。
- 沒有新增 00646 automatic provider。
- 沒有新增 S&P 500 自動抓取、匯率換算、database、login 或 scheduled job。
