# 資料格式說明

本文件說明 ETF Lookthrough Analyzer 目前支援的主要匯入 / 備份格式。

## 交易紀錄 CSV

交易匯入支援貼上 CSV、上傳 `.csv`，也支援從 Excel 複製的 tab-separated 表格。

### 欄位

支援繁體中文欄位：

```csv
日期,代號,名稱,類別,買賣,股數,成交價,手續費,交易稅,備註
```

也支援英文別名：

```csv
date,symbol,name,category,type,shares,price,fee,tax,note
```

### 買賣值

買進可填：

- `買進`
- `買`
- `Buy`
- `buy`
- `B`

賣出可填：

- `賣出`
- `賣`
- `Sell`
- `sell`
- `S`

### 範例

```csv
日期,代號,名稱,類別,買賣,股數,成交價,手續費,交易稅,備註
2026-05-20,0050,元大台灣50,台股核心 ETF,買進,10,190,1,0,範例
```

### 驗證規則

- 日期、代號、名稱、類別必填。
- 買賣必須可對應為買進或賣出。
- 股數必須大於 0。
- 成交價必須大於 0。
- 手續費與交易稅若有提供，必須大於或等於 0。
- 疑似重複交易的判斷條件是日期、代號、買賣、股數、成交價都相同。

## ETF 成分股匯入格式

ETF 成分股目前以手動貼上或 CSV 匯入為主。儲存時會取代同一檔 ETF 目前已儲存的成分股清單。

### 欄位

```csv
成分股代號,成分股名稱,權重,產業
```

### 範例

```csv
成分股代號,成分股名稱,權重,產業
2330,台積電,52.3,半導體
2317,鴻海,4.1,電子
2454,聯發科,3.8,半導體
```

### 匯入時另外設定

在 ETF 成分股頁面會另外填寫：

- ETF 代號
- 資料日期
- 資料來源

資料日期與來源只用於資料新鮮度 / 狀態標示，不做 ETF 經理人調倉研究。

## JSON 備份

JSON 備份用於保存整個 localStorage 中的 App 資料。

備份包含：

- 手動持股
- ETF 成分股
- 交易紀錄
- 手動價格資料
- App 設定

建議定期匯出 JSON 備份，尤其是在：

- 大量匯入交易紀錄前
- 大量更新 ETF 成分股前
- 清理瀏覽器資料前
- 更換電腦或瀏覽器前

匯入 JSON 備份後，通常需要重新整理頁面，讓 App 重新讀取 localStorage。
## 每日價格 CSV 格式

每日價格匯入可使用 `.csv` 檔案，或從 Excel / Google Sheets 複製後貼上 tab-separated 表格資料。

### 必要欄位

標題列為必填，至少需要：

- `日期` / `date`
- `代號` / `symbol`
- `收盤價` / `close` / `price`

### 可選欄位

- `名稱` / `name`
- `來源` / `source`
- `備註` / `note`

### 範例

```csv
日期,代號,名稱,收盤價,來源,備註
2026-05-20,0050,元大台灣50,190.5,手動整理,範例
```

### 驗證規則

- 日期為必填。
- 代號為必填。
- 收盤價必須大於 0。
- 名稱、來源、備註可留空。
- 無效列會保留在預覽中並顯示錯誤，不會被靜默丟棄。

### 重複價格處理

同一 `日期 + 代號` 已存在時，預覽會標示為同日同代號價格。

- 若勾選「覆蓋同日期同代號價格」，匯入時會更新既有價格。
- 若取消勾選，匯入時會略過重複價格。

匯入的每日價格會標記為 `sourceType: "csv"`，來源預設為 `CSV 匯入`，並記錄匯入時間。
